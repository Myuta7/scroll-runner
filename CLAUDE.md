# ScrollRunner — プロジェクトルール

YouTube Playable 向けのゲーム作成プロジェクト。

## 進捗記録ルール（最重要）

**作業をしたら、必ず `Progress.md` に進捗を記録すること。**

- コード追加・修正、ファイル生成、設定変更、公開作業など、何らかの作業を行うたびに `Progress.md` へ追記する。
- 各エントリには「日付 / 作業内容 / 変更ファイル / 次にやること」を含める。
- 新しい情報が出てきたら古い記述を更新し、最新状態を保つ。
- セッションを開始したら、まず `Progress.md` を読んで前回までの状況を把握してから作業を始める。

## プロジェクト概要

- **ゲームジャンル**: スクロールランナー（ScrollRunner）
- **配信先**: YouTube Playable
- **企画書とプロトタイプ**: 別セッションで生成中。生成後にこのフォルダへ配置される。
- **このセッションの役割**: 届いた企画書・プロトタイプをブラッシュアップし、GitHub へアップロードして GitHub Pages でテスト公開する。

## ワークフロー

1. 企画書・プロトタイプの受領・内容確認
2. ブラッシュアップ（ゲーム性・パフォーマンス・見た目の改善）
3. GitHub リポジトリへアップロード
4. GitHub Pages でテスト公開
5. 各ステップ完了ごとに `Progress.md` へ記録

## YouTube Playable 向けの注意点

- 単一 HTML／軽量構成を基本とし、外部依存を最小化する。
- モバイル縦画面・タッチ操作を前提にレスポンシブ対応する。
- ロード時間と初回描画を軽くする（アセットのインライン化・圧縮を検討）。
